Tables

Tables can be made by adding multiple 'td' elements (table data) in multiple 'tr' elements (table row) in a 'table' element. A 'th' element can be used in a row to define table headers, which are bold and centred but this can be changed through CSS, along with borders and other things. A caption can be added using the 'caption' element and its style can be modified

Properties 'border-collapse' and 'border-radius' can be modified to have single/rounded borders. Borders can be solid, but also dotted, dashed, double, groove, ridge, inset, outset and none/hidden. This, along with the colour, can be set in the 'border' property, but can also be their own 'border-color' and 'border-style' property, which has higher priority.

To modify the width of a table, you can add the style attribute to it and change the width. Also, you can add the style attribute and change the width of a 'th' or the height of a 'tr'. Measurements are in % screen or px

You can addjust the padding inside cells and the spacing between cells. You can merge cells using colspan and rowspan, where the content of the cell that is merged remains undefined.

You can create a zebra effect using the :nth-child() selector, putting it on 'td'/'th' makes it vertical and putting it on 'tr' makes it horizontal. You can overlap the zebra effect if you use a translucent colour. If you use 'border-bottom' on 'tr's instead of having a border around the whole table, you can have a horizontal dividers. If you use the ':hover' selector on 'tr's, you can have the row highlighted upon hovering.

you can specify colgroups to have columns styled a certain way, although only the width, visibilty, background and border can be changed. These must be placed right after the 'table' element, or right after the 'caption' element if it is present.

Display Values, Div and Span

Every element has a default display value, most commonly either block or inline.

Block-level elements always start on a new line, has some space (margin) before and after the element, and takes up the full width available. Examples include: blockquotes, h1-6, horiontal rules, lists, div and p.

Inline elements do not start on new lines and only take the width it needs. Examples include: a, q, button, img and span.

The div element is used as a containter for other HTML elements and commonly uses the attributes style, class and id. It can group sections and with CSS, it can be used to style blocks of content. Div is a block-level element, by default, it will take up the full width available. However, if the content doesn't take up the full width, you can specify the width then set the margin to auto. You can have div elements side by side on a page in a few ways, like using float, inline-block and grid, but flex is the best way from my research for simplicity

Classes and IDs

Classes are used to specify certain parts of a document to be used by CSS and JS. Classes can be used for many HTML elements, and one element can have many classes (by having a space in between them). Classes are referenced using a dot before the name. IDs are similar to classes, but only one element can have a certain ID and, when referenced, a # is used before the name. Names are case sensitive to classes and IDs.
